### CTA 设计（软植入，不硬广）

```

  绝不在文中间插广告，绝不用"震惊/颠覆/月入X万"等诱导话术。

三种 CTA 模式（轮换使用，同一天不重复）：

   具体怎么做的，我下篇详细拆。（关注不迷路）"

C. 社群邀请（最直接，少用）
  "最近在搞一个 AI 落地交流群，
   都是月营收 10 万以上的小老板在互相交流踩坑经验。
   想进的评论区说一下你的行业，我拉你。"

❌ 禁止：
  - 文中间突然插广告
  - "限时优惠""倒计时""不买就亏了"
  - 承诺收入（"学了就能月入 X 万"）
  - 贬低竞品
```

## Step 2: 选题筛选

### 核心筛选逻辑

**不是什么 AI 新闻都写，只写能打中小老板痛点的。**

每个候选按 4 维度打分（1-5 分）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 受众相关度 | ×3 | 小团队老板看了会觉得"跟我有关" |
| 实战价值 | ×2 | 读完能立刻动手（操作步骤/省钱方案/工具推荐） |
| 时效性 | ×1 | 24h 内热点 > 一周内 > 过期 |
| 情绪共鸣 | ×1 | "终于有人说人话了" / "原来还能这样" |

**总分 > 12 才入选**。纯技术新闻（如某模型发了论文）如果跟老板落地无关就不选。

### 5 篇选题配比

```
每天必须覆盖：
  1 篇 [工具实测]  → 亲自下场用，给出"小团队能不能用"的结论
  1 篇 [落地案例]  → 真实公司/个人用 AI 省了多少钱/赚了多少钱
  1 篇 [认知升级]  → 帮老板建立 AI 思维框架（如"先替代重复工作"）
  2 篇 [灵活搭配]  → 热点评论/行业分析/工具对比/趋势判断

配比铁律：
  ├ 至少 3 篇跟"AI 如何帮小团队省钱/赚钱"直接相关
  ├ 禁止 5 篇全是纯工具测评（读者疲劳）
  ├ 禁止 2 篇以上纯新闻转述
  └ 跟前 3 天选题重复率 < 30%
```

### 选题角度转换（关键技巧）

```
原始热点 → 小老板视角

❌ "DeepSeek V4 发布，万亿参数"
✅ "DeepSeek V4 免费用了，小团队可以拿它干这 3 件事"

❌ "AI Agent 市场规模将达 XXX 亿"
✅ "我用 AI Agent 替代了半个客服，一个月省了 6000 块"

❌ "斯坦福 AI 报告：美国投资 2859 亿"
✅ "斯坦福报告里藏着一个信号：不会用 AI 的小公司要被淘汰了"
```

## Step 4: 文案生成（卡兹克风格 + 训练营植入）

### 风格规则

完整规则见：`references/khazix-style-rules.md`

### 5 个核心风格锚点

```
1. 活人感优先：口语化、自嘲、情绪标点，像跟朋友聊天
2. 亲自下场：工具必须具体名字，案例必须真实发生的
3. 波动式节奏：长短句交替，一句话独立成段，疑问句做刹车
4. 知识随手掏：引用是"聊着聊着想起来的"，不是"下面科普"
5. 文化升维：从具体事件连接到更大的商业/哲学洞察
```

### 文案结构模板

```
─── 开头（200-400字）─── 四选一：
  A. 叙事启动：「故事是这样的。{具体场景}」
  B. 荒诞事实：「{直接抛出让人"？？？"的事实}」
  C. 热点破题：「{最近被xxx刷屏了}」
  D. 好奇心驱动：「{最近发现了一个xxx，很有意思}」

─── 正文（1500-3000字）───
  [核心论述] 工具实测/数据分析/案例拆解
    → 始终带着"小团队老板怎么用"的视角
  [知识随手掏] 自然插入 1-2 个商业/技术洞察
  [独立判断] 「坦率的讲...」「我自己的判断是...」

─── 结尾（200-400字）───
  [观点收束] 用一个短句/引用收住
  [CTA 软植入] 从 A/B/C 三种模式选一种，自然接在结尾后面

─── 固定尾部 ───
  以上，觉得有启发的话，点个「在看」分享给更多人。
  有问题或者想交流的，评论区见。
```

### 文案质量标准

- **字数**：2000-4000 字
- **具体名字**：工具名/公司名/人名，必须准确
- **数据引用**：每篇至少 2 个可验证的数据点
- **受众锚定**：每篇至少 3 处直接对"小团队老板"说话
- **独立观点**：每篇至少 1 个明确的个人判断
- **口语化词组**：全文至少 8 个不同的
- **CTA**：每篇 1 个，放结尾，不能生硬

### 禁忌清单

```
[风格禁忌]
  禁用词：说白了、本质上、综上所述、值得注意的是
  禁用标点：冒号「：」→逗号、破折号「——」→逗号句号
  禁用结构：bullet point > 3 个、大段加粗、宏大叙事开场
  禁用开头：「在当今AI时代」「随着技术的发展」

[引流禁忌]
  禁止：承诺收入、贬低竞品、夸大效果、限时逼单
  禁止：文中间插广告、连续两篇用同一种 CTA
  禁止：把文章写成广告软文（干货比例必须 > 90%）
```

### 四层自检

```
L1 硬性规则：禁用词 + 禁用标点 + 套话扫描 + 工具名检查
L2 风格一致：开头模式 + 节奏波动 + 口语化密度（≥8）
L3 内容质量：观点支撑 + 数据准确 + 受众相关度
L4 活人感终审：温度感 + 独特性 + CTA 自然度
L5 引流合规：CTA ≤ 1 个/篇 + 干货 > 90% + 无违禁话术
```

## ⚠️ Pitfalls

1. **干货 > 90%**：一旦读者觉得"这是广告"，整篇文章的信任度归零
2. **CTA 必须自然**：从内容自然过渡，不能突然冒出来
3. **不承诺收入**：「学了月入X万」是虚假宣传，也是训练营红线
4. **数据必须真实**：编造数据会被读者扒，毁掉所有信任
5. **受众精准**：不是给程序员看的，是给不懂技术的小老板看的，话要说人话
6. **5 篇/天量大**：质量不够宁可减量，水文比不发更伤品牌
7. **CTA 轮换**：连续 2 天用同一种 CTA 读者会脱敏
